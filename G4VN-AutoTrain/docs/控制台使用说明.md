# 控制台使用说明

## 控制台窗口

我已经修改了程序，现在会在启动时自动显示一个**控制台窗口**（黑色的命令行窗口）。

## 如何查看控制台

### 方法1：自动显示（推荐）

重新编译并运行程序后，会自动弹出一个黑色的控制台窗口，显示所有调试信息。

### 方法2：在Visual Studio中查看

如果您在Visual Studio中运行程序（按F5调试）：
1. 查看 **"输出"** 窗口
2. 在输出窗口的下拉菜单中选择 **"调试"** 或 **"显示输出来源: 调试"**
3. 所有 `Console.WriteLine` 的输出都会显示在这里

### 方法3：查看日志文件（如果添加了日志功能）

如果程序将输出写入文件，可以在程序目录中查找日志文件。

## 控制台会显示什么

控制台会显示以下信息：

### 启动信息
```
=== G4VN自动挂机启动器 ===
控制台已启用，可以查看调试信息
```


### 破解过程信息
```
DEBUG: Step 0 - Initial Setup

```

## 如果看不到控制台窗口

如果运行程序后没有看到控制台窗口，可能的原因：

1. **程序以GUI模式运行**：某些情况下控制台可能被隐藏
2. **被其他窗口遮挡**：检查任务栏，可能有控制台窗口在后台

## 解决方案

如果控制台窗口没有显示，可以：

1. **检查任务栏**：查看是否有命令行窗口的图标
2. **使用Alt+Tab**：切换窗口，找到控制台窗口
3. **在Visual Studio中运行**：按F5调试，查看"输出"窗口

## 关闭控制台窗口

控制台窗口会随着程序一起关闭。如果程序在后台运行，控制台窗口也会在后台。
